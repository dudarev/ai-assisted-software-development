# Twitter Thread: Weekly YouTube Picks (2026-W09)

My AI coding YouTube picks this week are about making coding agents truly operational.

A few good ideas:

Pick the right harness boundaries.

Raise your verification frequency.

Treat context and guardrails as first-class artifacts.

🧵👇

---

1️⃣ Lada Kesseler at DevCon Fall 2025: "Emerging Patterns for Coding with Generative AI"

The durable shift is from “better prompting” to context management. Use reloadable knowledge docs and "specialist agents". Love the “feedback flip” to force a checker pass! 🔄

🔗 https://www.youtube.com/watch?v=M-zOSEJFNos

---

2️⃣ @davefarley77 breaks down "How to Make the Best of AI Programming Assistants".

AI increases change frequency, so teams often under-sample their own output. The fix? Faster signals, smaller diffs, and running CI on every meaningful step. Shrink the batch size! 📉

🔗 https://www.youtube.com/watch?v=XavrebMKH2A

---

3️⃣ Pi Coding Agent shows the power of open-source harness customization.

It’s not about smarter prompts—it’s about harness leverage! Hooks, UI widgets, and a custom event loop let you enforce tasks and block dangerous commands. 🛠️

🔗 https://www.youtube.com/watch?v=f8cfH5XX-XU

---

4️⃣ OpenClaw vs Claude Code (@AnthropicAI): Reach vs Control. 

A practical decision axis: always-on reach gives you proactive help but a bigger blast radius. Repo-scoped code loops trade some access for safer boundaries. Isolate the high-reach agent! 🛡️

🔗 https://www.youtube.com/watch?v=9f38h_GooA8

---

5️⃣ Dru Knox on "Context as Code — Stop Prompting, Start Engineering".

Context needs CI! Treat it like software: validate formats, run scenario-based evals, and mine logs. Without a lifecycle and update mechanism, context just rots. 🏗️

🔗 https://www.youtube.com/watch?v=TlC7jq4ooSM
