## Summary

The discussion examines Dario Amodei's (CEO of Anthropic) claim that AI will write 90-100% of code within a year. Igor Kotenkov argues that while the quantity of AI-generated code will surge, the role of the programmer is not disappearing but evolving. The conversation shifts focus from "writing code" to "system engineering," where programmers manage architecture, security, and business requirements while delegating implementation to AI. Using data from Google (25% AI-written code) and tools like Aider (80% AI-written in specific cases), the participants explore how infrastructure, economic paradoxes (Jevons Paradox), and iterative feedback loops define the new era of "vibe coding" and AI-assisted development.

## Key points

- **Literal vs. Practical 90%**: Dario's 90% claim might refer to solving specific benchmarks or internal routine tasks rather than replacing all human coding effort.
- **Infrastructure Multiplier**: A model's effectiveness is heavily dependent on the surrounding system (context engineering, RAG, IDE integration). Raw chat is significantly less productive.
- **Role Shift**: Programmers are becoming "AI Intern Managers" or "Autocrats," focusing on high-level specs while AI handles the "dumb" task of syntax generation.
- **Jevons Paradox in Coding**: Making code 10x cheaper to produce will likely lead to a 20x-100x increase in demand for software, potentially hiring *more* developers to manage the complexity.
- **Context is King**: 65% of GitHub code is in 5-6 languages. Models are exponentially better at these "popular" languages due to training data density.
- **Iterative Debugging**: Most complex AI coding tasks require 2-3 iterations of human feedback to reach production quality.

## Concepts / principles

**Comparative Advantage (Economy)**
Even if a senior developer is better at everything than an AI, it is economically beneficial to delegate routine tasks (tests, documentation, boilerplate) to the AI to maximize the developer's output on high-leverage architectural tasks.

**Jevons Paradox**
In economics, the Jevons paradox occurs when technological progress or government policy increases the efficiency with which a resource is used (reducing the amount necessary for any one use), but the falling cost of use induces so much more use that it increases the total consumption of that resource. Applied to AI, cheaper code leads to more software being built, not fewer developers.

**Infrastructure vs. Model**
A high-performance model (like a Lamborghini) is useless in an environment without roads or gas stations. Modern AI coding requires the "roads" of IDE integration, repository-wide context, and automated testing to be effective.

## Patterns

- **The "AI Intern" Pattern**: Treating the AI as a junior developer that needs precise specifications and code reviews.
- **Vibe Coding**: High-level, often voice-driven or natural language-driven development where the human supervises the "vibe" and results rather than the syntax.
- **Self-Developing Tools**: Using tools like Aider to write the code for Aider itself, creating a virtuous cycle of productivity.
